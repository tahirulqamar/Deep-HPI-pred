Overview

Deep-HPI-pred is a robust web-based application designed to aid researchers in studying host-pathogen interactions using advanced computational approaches. By integrating state-of-the-art deep learning algorithms with an intuitive graphical user interface, it provides a comprehensive platform for both detection and visualization of host-pathogen interaction networks.

Key Features

  • Detect Host-Pathogen Interaction Networks: Our application uses advanced deep learning algorithms to predict host-pathogen interactions with high accuracy. Upload your data, and Deep-HPI-pred will analyze it, detecting potential interaction networks based on the information provided.
  • Visualize Interaction Networks: Deep-HPI-pred doesn't just detect interaction networks; it also provides tools to visualize them. This allows researchers to better understand the complexities of host-pathogen interactions.
  • Generate Hypotheses: By using Deep-HPI-pred, researchers can generate meaningful hypotheses for their studies. These can then be tested in a lab setting, driving new discoveries and breakthroughs.
  • Design Experiments: In addition to hypothesis generation, our tool can also assist in designing appropriate experiments. By providing a clear view of the host-pathogen interaction network, it helps researchers plan effective and efficient experimental designs.
  • Regular Updates and Improvements: We are continuously working to enhance our tool's accuracy and usability. Our users will benefit from ongoing updates and improvements to ensure they are always using the best possible prediction tool.

Intended Users

Deep-HPI-pred is designed to serve both experimental biologists studying host-pathogen interactions in a lab setting and bioinformaticians interested in computational approaches. The straightforward user interface ensures ease of use for all, regardless of computational expertise.

User Interface

The app's UI contains several tab panels, each serving a distinct purpose:

  • Home: This panel provides an introduction to the application, and quick links to the app's GitHub repository, documentation, FAQ, and contact information.
  • Browse Search: In this panel, users can upload their own data or load demonstration data. The file format for uploading data is CSV and it should include UNIPROTKB identifiers.
  • Results: This panel displays the network visualization of the predicted interactions along with a downloadable data table of results.
  • GO Analysis: In this panel, a Gene Ontology (GO) enrichment analysis is performed, and the results can be visualized in plot.

Server Logic

  • The server logic primarily revolves around handling user input and rendering results.
  • User file uploads are handled, ensuring data is properly loaded into the application.
  • The application also provides demonstration data that can be loaded with a button click.
  • Users can train the model and predict interactions with their data or the demonstration data.
  • The predicted interactions are then rendered as network visualizations.
  • GO enrichment analysis is performed and results are displayed as different plot types, which can be selected by the user.

Network Visualizations

Network visualizations are created using the visNetwork package, with nodes representing proteins (colored differently based on whether they are host or pathogen proteins) and edges representing interactions between these proteins.

GO Enrichment Analysis

GO enrichment analysis provides a means of interpreting the functional aspects of the results. It is performed on the predicted interactions, and the results can be visualized in various ways.

Note

All results, including tables and plots, can be downloaded for further analysis or reporting purposes.

Libraries

This application uses numerous libraries, including but not limited to:

  • shiny for building the web app.
  • visNetwork for network visualization.
  • igraph, tidygraph and ggraph for network analysis and visualization.
  • dplyr and tidyverse for data manipulation.
  • ggplot2 for data visualization.
  • magrittr for piping (%>%) functions.
  • httr and XML for handling HTTP requests and XML data.
  • tidyr for data tidying.
  • shinythemes for shiny themes.
  • DOSE for disease ontology semantic and enrichment analysis.
